# 화랑 — 학습 관리 웹앱

학생이 매일 공부한 내용을 자유 서술형으로 기록하고, 선생님이 과목별 완료 퍼센트를 매겨
막대그래프로 보여주는 웹앱. React(Vite) + Firestore. 별도 백엔드 서버 없이 프론트에서
Firestore를 직접 읽고 씁니다.

## 역할 3가지

- **학생**: 이름 + 4자리 코드로 로그인. "일상 공부" 메뉴는 3개로 나뉩니다.
  - **금일 학습 계획**: 오늘 하려는 공부 계획을 적는 곳(참고용, 채점 대상 아님)
  - **금일 학습량**: 오늘 실제로 한 학습량을 적는 곳(선생님이 채점)
  - **점검**: 날짜별로 과목 박스+그래프를 확인. 각 과목 박스에는 그날 계획으로 적었던
    내용도 참고 텍스트로 같이 표시됩니다.
- **학부모**: 자녀와 같은 이름 + 별도 코드로 로그인. 자녀의 "점검" 화면만(날짜 목록 →
  상세) 읽기 전용으로 볼 수 있고, 선생님이 남긴 코멘트도 여기서 보입니다.
- **선생님**: 코드로 로그인. 학생별 "점검" 화면에서 과목별 퍼센트를 입력하고, 학부모에게만
  공개되는(학생에게는 비공개) 코멘트도 남길 수 있습니다.

## 기술 스택

- React (Vite) + react-router-dom
- Firebase Firestore (Firebase Auth는 사용하지 않음 — 코드 로그인은 Firestore 조회로 처리)
- recharts + CSS 기반 미니 막대(meter)
- vitest (파싱/계산 로직 단위 테스트)

## 로컬 개발

```bash
npm install
npm run dev
```

```bash
npm test        # parseSubjects / computeOverallPercent 단위 테스트
npm run build   # 프로덕션 빌드
```

## Firebase 프로젝트 설정 (아직 안 했다면)

1. [Firebase 콘솔](https://console.firebase.google.com)에서 새 프로젝트 생성
2. Firestore Database 활성화 (프로덕션 모드로 시작해도 무방 — 규칙은 이 저장소의
   `firestore.rules`로 덮어씀)
3. 프로젝트 설정 → 일반 → "웹 앱 추가" 후 나오는 config 값을 `.env`에 채우기
   (`.env.example`을 복사해서 `.env` 생성)
4. `.firebaserc`의 프로젝트 ID를 실제 프로젝트 ID로 교체

## 데모 계정 시딩 (Firestore 콘솔에서 수동으로)

`students` 컬렉션에 문서 하나:

```json
{ "name": "지창", "code": "2585" }
```

`teachers` 컬렉션에 문서 하나:

```json
{ "code": "880603" }
```

`parents` 컬렉션에 문서 하나 (`studentId`는 위 `students` 문서의 자동 생성 ID):

```json
{ "name": "지창", "code": "0800", "studentId": "<students 문서 ID>" }
```

(문서 ID는 전부 자동 생성 ID로 두면 됩니다. `students` 문서의 ID가 로그인 후 사용되는
`studentId`이고, `dailyLogs` 문서 ID의 접두어로도 쓰입니다.)

## 배포

```bash
npx firebase login
npx firebase deploy --only firestore:rules,hosting
```

`npm run build`로 만든 `dist/`를 Hosting이 서빙합니다(`firebase.json` 설정 완료).

## 알려진 한계

- **인증 없음**: Firebase Auth를 쓰지 않기로 한 설계라 Firestore 보안 규칙이 요청자를
  검증할 수 없습니다. 배포 URL과 Firebase 웹 config를 아는 사람은 누구나 데이터를 읽고
  쓸 수 있습니다. 소수 인원이 쓰는 데모용 트레이드오프이며, 더 넓게 쓰려면 Firebase
  Auth 도입이 필요합니다 (`firestore.rules` 참고).
- **선생님 코멘트는 UI에서만 숨겨집니다**: 학생에게 코멘트를 안 보여주는 건 화면
  컴포넌트가 렌더링하지 않는 방식으로 구현되어 있습니다. 인증이 없어 `dailyLogs` 문서
  자체는 누구나(학생 포함) 읽을 수 있으므로, 브라우저 개발자 도구로 Firestore를 직접
  호출하면 학생도 이론적으로 코멘트를 볼 수 있습니다. 진짜 접근 제어가 필요하면
  Firebase Auth + 보안 규칙으로 다시 설계해야 합니다.
- **과목 파싱**: 과목 키워드가 문장 어디에 등장하든(문장 맨 앞이 아니어도) 그 위치부터
  다음 과목 키워드 전까지를 해당 과목 내용으로 인식하는 단순 정규식 방식입니다. 다른
  과목을 설명하는 문장 중간에 우연히 과목 키워드가 섞여 있으면 잘못 나뉠 수 있습니다.

## 검증된 것 / 직접 확인이 필요한 것

로그인, 계획/학습량 저장(같은 날짜 문서에 병합 저장되어 서로 안 지워지는지), 선생님
퍼센트·코멘트 저장, 학부모 로그인·조회는 실제 Firebase 프로젝트에 대해 스크립트로
확인했습니다. 다만 화면(UI)에서 직접 클릭해보며 검증한 것은 아니므로, 아래는 브라우저에서
한 번씩 확인해보시는 걸 권장합니다.

- [ ] 학생 코드(이름 "지창"/코드 `2585`)로 로그인
- [ ] 학부모 코드(이름 "지창"/코드 `0800`)로 로그인 → 자녀 날짜 목록만 보이는지
- [ ] 선생님 코드(`880603`)로 로그인
- [ ] 학생이 "금일 학습 계획"과 "금일 학습량"에 각각 다른 내용을 저장 → 서로 안 섞이는지
- [ ] "점검"에서 과목 박스 아래에 그날 계획 텍스트가 참고용으로 뜨는지
- [ ] 선생님이 퍼센트 + 코멘트 저장 → 학생 화면엔 코멘트가 안 보이고, 학부모 화면엔 보이는지
